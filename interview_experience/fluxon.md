# Interview Questions – UI / System Design Round

---

## UX Collaboration & Design Process

1. What does it look like when you give requirements to the UX team?
2. Do you provide requirements as documents, Jira tickets, or something else?
3. Once designs are in Figma, are they finalized or is there back-and-forth?
4. Who decides what fields, modals, and UI components should be shown?
5. Can you describe the best designer you’ve worked with and what made them effective?
6. What can a designer do to make your job easier?
7. What do you do to make a designer’s job easier?
8. How do you handle situations where designers introduce components not in the existing system?
9. How do you decide whether to push back or add a new component to Storybook?
10. Are you familiar with design systems?
11. How do you ensure consistency across multiple screens in a large application?
12. How do you manage and maintain common components across projects?
13. Are you familiar with accessibility standards?
14. Have you implemented accessibility standards in production?
15. Are you familiar with animations and micro-interactions?
16. How do you analyze and understand complex animated websites?
17. How do you think smooth transitions and animations are implemented?

---

## JavaScript / Coding

18. What will be the output of a `for` loop with `var` and `setTimeout(..., 0)`?
19. Why does the value print the same number multiple times?
20. How does the event loop and callback queue affect execution order?

---

## System Design – Admin Dashboard (Ride Sharing Platform)

21. How would you design a scalable internal admin dashboard?
22. How would you show near real-time trip and driver metrics?
23. What should load first — summary metrics or detailed tables?
24. What are the trade-offs between loading small vs large datasets initially?
25. How would you structure routing for this dashboard?
26. How would you handle 500K driver records?
27. How would you implement pagination?
28. What pagination technique would you use?
29. What are the drawbacks of fixed-size pagination?
30. What is the difference between offset-based and cursor-based pagination?
31. If 100 records are fetched but only 30 fit on screen, how would you render them?
32. Would you render all records at once or lazily load?
33. What happens to previously rendered page data when navigating across pages?
34. How would you cache paginated data?
35. Would you use client-side cache or server-side cache?
36. How would you implement filtering (e.g., rating > 4.5)?
37. Should filtering be done on frontend or backend?
38. What should happen to pagination when a filter is applied?
39. If you are on page 5 and apply a filter, which page should you land on?
40. What happens if the same dashboard is opened in multiple tabs?
41. Will multiple tabs interfere with each other?
42. What kind of performance impact would multiple tabs cause?

---

## React Performance & Rendering

43. How would you implement a light/dark theme toggle?
44. How would you avoid unnecessary re-renders?
45. How does React re-rendering work?
46. How can you prevent child components from re-rendering when parent updates?
47. Is `useRef` a way to prevent re-rendering?
48. What other techniques exist to optimize rendering?
49. How does state change trigger re-render?
50. How do you manage large tables efficiently in React?

---

## Scalability & Performance

51. If traffic increases 100x, what will break first?
52. Where would bottlenecks occur — network, server, or frontend?
53. How would you handle high load?
54. Would load balancers help?
55. How would debouncing or throttling help?
56. How would you reduce unnecessary API calls?

---

## Career / Role Fit

57. Is this role frontend-only or full-stack?
58. Will there be opportunities to contribute to backend development?
59. How flexible is the tech stack exposure?
